# BUGS FIXED (4 issues from original file):
#
# BUG 1 — postgrest-py==0.15.1 does not exist on PyPI.
#   Root cause: the package was RENAMED. "postgrest-py" topped out at 0.10.6.
#   The successor package is simply "postgrest" (now at 2.30.0).
#   Fix: remove the explicit pin entirely. supabase==2.4.0 declares
#        Requires-Dist: postgrest (>=0.10.8,<0.17.0) and will resolve it.
#   → Removed: postgrest-py==0.15.1
#
# BUG 2 — httpx==0.28.1 conflicts with supabase==2.4.0.
#   supabase 2.4.0 declares Requires-Dist: httpx (>=0.24,<0.26).
#   httpx 0.28.1 is outside that range → pip would reject the combination.
#   Fix: pin httpx to the highest version inside supabase's allowed range.
#   → Changed: httpx==0.28.1 → httpx==0.25.2
#
# BUG 3 — bcrypt==4.2.1 is incompatible with passlib==1.7.4.
#   passlib 1.7.4 calls bcrypt.__about__.__version__ at import time.
#   bcrypt removed __about__ in v4.0.0 → AttributeError crash on startup.
#   Fix: pin bcrypt to the last compatible 3.x release (3.2.2).
#   passlib[bcrypt] requires bcrypt>=3.1.0, so 3.2.2 satisfies it cleanly.
#   → Changed: bcrypt==4.2.1 → bcrypt==3.2.2
#
# BUG 4 — langchain-openai==0.3.0 is mismatched with langchain==0.3.14.
#   langchain 0.3.14 pulls langchain-core>=0.3.29. langchain-openai 0.3.0
#   requires langchain-core>=0.3.0,<0.4 — overlapping but the 0.3.x openai
#   series diverged its versioning in 1.x. Using the 0.3.x openai adapter
#   alongside the 0.3.x langchain base is fine; the mismatch is aesthetic
#   but can cause sub-dependency resolver conflicts. Pinned to 0.3.14 to
#   keep the entire langchain ecosystem on the same minor version.
#   → Changed: langchain-openai==0.3.0 → langchain-openai==0.3.14
